# Review Handoff

## Agent 提交人工审查前必须提供

1. **变更摘要**: 修改了哪些文件、为什么修改
2. **验证结果**: 执行了哪些检查、结果是什么
3. **跳过的检查**: 如果有检查未执行，说明原因
4. **残留风险**: 变更可能影响的其他区域
5. **未解决问题**: 发现但未处理的问题

## 审查者关注点

- 技能 SKILL.md 的触发条件是否准确覆盖目标场景
- 工作流步骤是否可执行（命令存在、路径正确、信号明确）
- 是否有猜测性内容被写成事实
- Bundled package 版本引用是否与 `bundled-packages.json` 一致
- 修改范围是否超出需求

## 需要升级到人工处理的场景

- 修改 `bundled-packages.json` 中的版本或来源
- 删除或重命名已有技能
- 修改 `.claude/settings.json` 的权限配置
- 修改 git hooks
- 涉及多个技能的交叉修改
- 任何不确定是否安全的操作
